About style effects

Style effects are alternative widget styles that are automatically applied to a widget while it's in a particular state in the web browser, such as while being moused over or clicked. When the widget enters the triggering state, the style effect is applied. When the widget exits the triggering state, the style effect is removed and the widget reverts back to its base styling.

Read more on axure.com

Style effects in this library

Style effects were added to widgets based on the Carbon Design System color guidelines and component specifications.

:mouseover   (:hover) and :mousedow   (:active) styles model mouse interactions.

:mousedown

:disabled      styles change the appearance of the widget when its state is changed by an event, or can be used to display elements as disabled by default.

Editing components

Some components are constructed of multiple shapes, but still need to react to mouse interactions at the same time as one element (e.g. buttons with icons, table rows etc.).

For these groups, the "Fire Mouse Style Effects” is enabled in the Interactions pane.

In any case when such a group is ungrouped during the design process, make sure to group the elements again and enable the above setting.
